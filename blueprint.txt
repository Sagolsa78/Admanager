1. Project Overview
AdForge is an AI-powered Brand DNA analysis platform. Users submit a website URL and a LinkedIn profile URL; the backend scrapes and analyses both, then returns 5 distinct brand positioning contexts. Users rate, edit, provide feedback, re-analyse, export, and save their results to a personal dashboard that doubles as an ad campaign manager.

3. Pages — Detailed Specifications
3.1  Landing Page  /
The landing page is the primary conversion surface. Dark background, orange accent energy matching the AdForge reference screenshot but with improved layout density and motion.
Sections (top to bottom):
Navbar — Logo left, links centre (Features, Templates, Pricing), Sign In + Start Free right. Sticky on scroll, background blurs via backdrop-filter. On mobile: hamburger drawer.
Hero — Full-viewport height. Left: eyebrow badge (NOW IN PUBLIC BETA), H1 headline with orange accent words, subtext paragraph, two CTA buttons (Get Started Free → /sign-up, Watch Demo → modal). Right: animated product UI mockup (the calendar/post-queue screenshot, replicated as a live HTML/CSS component, not an image).
Social Proof Bar — Scrolling marquee of logos or a row of 3 stat counters: 30+ Posts Generated · All Content Scheduled · 100% Automated.
How It Works — 3-step horizontal flow: Enter URLs → AI Analyses → 5 Contexts Ready. Each step has an icon, number pill, short description. Animated connector line between steps.
Features Grid — 2x3 card grid. Each card: icon, bold title, 2-line description. Key features: Brand DNA Analysis, 5 Context Variations, LinkedIn + Web Scraping, Rating & Feedback Loop, One-Click Export, Dashboard History.
Context Preview — Animated carousel or tab switcher showing a sample of what a generated context card looks like. Gives users a taste of the output before signing up.
Pricing Teaser — 3 pricing tiers side by side (Free / Pro / Agency). Highlight the Pro tier. Each card lists features as bullet points. CTA per card. Link to full /pricing page.
Footer — Logo, nav links, social icons, copyright. Two-column layout.

Deeper narrative on what AdForge does. Target: users who landed on the site and want to understand capabilities before signing up.
Hero banner — H1 + short paragraph, single CTA.

3.5  Analyse Page  /analyse
This is the core product experience. The most complex page. It has 4 distinct phases that are rendered in place (no page navigation between phases).
Phase 1 — Input Form
Centred card, max-width 640px
H2: 'Analyse Your Brand DNA'
Input 1: Website URL — label, placeholder 'https://yourwebsite.com', validation for valid URL
Input 2: LinkedIn Profile URL — label, placeholder 'https://linkedin.com/in/yourhandle', validation
Large CTA button: 'Analyse Now →' — full width, orange
On submit: validates both URLs with Zod, then fires POST /api/analyse, transitions to Phase 2
Phase 2 — Loading / Lazy Analysis
Replace form with a full-screen loading experience (not a spinner — make it visual)
Animated 'orb' or pulsing ring in centre of screen
Status messages cycle every 2–3 seconds: 'Scanning your website...' → 'Reading LinkedIn profile...' → 'Extracting brand signals...' → 'Mapping positioning angles...' → 'Generating 5 contexts...'
Progress bar or segmented dots showing completion stages
Poll GET /api/analyse?id={jobId} every 3 seconds until status === 'complete'
Estimated time label: 'Usually takes 20–40 seconds'
Phase 3 — 5 Context Cards
On completion: transition from loading orb to card grid with staggered fade-in (Framer Motion: each card delays by 150ms * index)
Layout: vertical stack on mobile, 2-column grid on desktop (with card 5 centred below)
Each Context Card contains:
Context number badge (e.g. '01') + context title
Full context body text (collapsible — show first 4 lines, 'Read more' expands)
Star rating row (1–5 stars) — user rates relevance of this context
Thumbs up / Thumbs down quick feedback
'Edit' icon button — opens inline textarea to refine the context text
'Save' button — saves this individual context to /saved
Phase 4 — Feedback & Actions Bar
Sticky bottom bar that appears after cards load
Left: 'Submit Feedback' — opens a modal with a textarea for overall feedback + submit button (POST /api/feedback)
Centre: 'Re-analyse' — clears state, returns to Phase 1 form
Right: 'Export PDF' — generates a PDF of all 5 contexts + ratings
Far right: 'Save All to Dashboard' — bulk saves all contexts

8. Animation Guidelines
All animations via Framer Motion. Keep them purposeful — every animation must communicate something.
Page transitions
Wrap each page in a <motion.div> with initial={{ opacity: 0, y: 8 }} animate={{ opacity: 1, y: 0 }} exit={{ opacity: 0, y: -8 }} transition={{ duration: 0.25 }}
Context cards stagger
Parent: variants with staggerChildren: 0.15. Child: variants fade+slide up.
Loading orb rings
3 rings: CSS keyframes scale 1→1.6 + opacity 1→0, staggered by 0.5s each.
Status text cycle
AnimatePresence with key={statusMessage}: exit slides up + fades, enter slides up from below.
Sidebar collapse
motion.div with animate={{ width: collapsed ? 64 : 240 }} transition={{ type: 'spring', stiffness: 300 }}
Button press
whileTap={{ scale: 0.97 }} on all interactive buttons.
Star rating
whileHover={{ scale: 1.2 }} on each star icon.
Card hover
whileHover={{ y: -2, boxShadow: '0 8px 32px rgba(232,80,10,0.12)' }}

10. Key Technical Decisions & Notes
10.1  Polling Strategy for Analysis
The /analyse page fires a POST to start the job, receives a jobId, then polls GET /api/analyse?jobId=xxx every 3 seconds using React Query's refetchInterval. When status === 'complete', React Query stops polling and the Zustand store transitions analysePhase to 'results'. This avoids WebSocket complexity while keeping the UX responsive.
10.2  PDF Export
Use jsPDF + html2canvas: render all 5 context cards to canvas, then embed in a PDF with the AdForge logo header. Alternatively, use a hidden <div> styled for print and trigger window.print() with a dedicated print stylesheet. jsPDF gives more control.
10.3  Kanban Drag & Drop
Use @dnd-kit/core — it is accessible, performant, and has no default styling. Optimistically update the campaign status in React Query cache on drop, then PATCH /api/campaigns/:id in the background. Roll back on error.
10.4  Mobile Responsiveness
All pages must be fully responsive. Dashboard sidebar collapses to a bottom tab bar on mobile (5 icons: Home, Analyse, Campaigns, Saved, Settings). Context cards stack to single column. Landing page hero stacks vertically on mobile with product mockup below the text.
10.5  Performance
Use next/image for all images.
All dashboard modules are loaded with dynamic(() => import(...), { ssr: false }) to keep the initial JS bundle small.
React Query caches all GET responses — no unnecessary refetches.
Tailwind purges unused classes — final CSS bundle under 20KB.
